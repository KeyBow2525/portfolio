Web Application / FastAPI + React

MediaMaster Async

教育現場で必要とされるメディア変換・ファイル配布をまとめて解決するWebアプリです。
HuggingFace Spaces上で動作し、ブラウザだけで変換処理を完結させます。

課題とコンセプト

  • ▸教員から要望が多数寄せられたメディア変換ニーズ（HEIC・M4A・MP4 など）を1か所にまとめたい
  • ▸変換処理が長時間になっても画面がフリーズしない、快適なUXの実現
  • ▸機能追加コストを最小化するモジュール設計（新機能を4ステップで追加できる構造）
  • ▸個人情報を含む配布ファイルを安全に管理・配信する仕組みの構築
  • ▸インフラ費用ゼロで運用できるプラットフォームの選定

工夫した点・技術的なこだわり

非同期処理とポーリングによる快適なUXの実現

メディア変換は処理時間が数秒〜数十秒に及ぶため、同期処理では画面がフリーズしてしまいます。FastAPIのBackgroundTasksを活用し、リクエスト受付後に即座にtask_idを返却、フロントエンドが定期的にステータスをポーリングする非同期アーキテクチャを採用しました。処理中もUIが操作可能な状態を保ち、完了次第ダウンロードリンクが表示される、ストレスのない体験を実現しています。

4ステップで機能追加できるモジュール設計

tools/配下に機能を独立したモジュールとして分離し、app.pyはルーティングのみを担当する設計にしました。新機能の追加は①ツールファイル作成、②tool_id登録、③分岐1行追加、④フロントカード追加の4ステップで完結します。教員から新たな変換ニーズが上がった際に、最小限のコストで対応できる拡張性を意識した設計です。

多層的なセキュリティ設計

1ファイルあたり100MBの上限チェック、asyncio.Semaphoreによる同時処理数の制限（最大3件）、tool_id・output_formatのバリデーションをすべてバックグラウンドタスクに到達する前に処理することで、堅牢なシステムを構築しました。また、ファイル配布機能ではos.path.basename()によるパストラバーサル対策も実施しています。

HuggingFace Datasetsを活用したセキュアなファイル配布

配布ファイルはGitリポジトリに含めず、privateリポジトリのHuggingFace Datasetsで管理しています。コンテナ起動時にstartup.pyがファイルを取得してローカルに配置する設計で、ファイルの更新は「Restart Space」のみで完結します。パスワードはHFS Secretsで管理し、フロントエンドには一切渡らないため、ソースコードを公開しても突破できない認証フローを実現しました。

新機能追加フロー

成果と反響

「iPhoneで撮った写真をパソコンで使えない」「授業の録音データをMP3にしたい」など、現場から上がっていた変換ニーズを1つのWebアプリに集約したことで、複数の業務課題を同時に解決しました。インストール不要でブラウザからアクセスするだけの手軽さが受け入れられ、実際に多くの教員から感謝の声が寄せられています。

インフラコストゼロのHuggingFace Spaces上での運用、ffmpeg・ImageMagickなどの外部ツールをDockerで自由にコントロールできる設計、モジュール化による拡張容易性など、技術的にも実践的なアーキテクチャ判断を数多く経験できたプロジェクトです。
